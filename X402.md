# x402 — pay-per-verification for the Warrant agent

Warrant's agent doesn't just *hold* money — it **charges for its work**. The
agent's AI verification is exposed as an **[x402](https://www.x402.org/)-metered
service**: each proof submission pays a small **cUSD micropayment to the agent's
wallet** over the x402 protocol (HTTP `402 Payment Required` → wallet-signed
stablecoin authorization → on-chain settlement), auto-paid from the claimant's
connected wallet. Combined with the agent's **ERC-8004 identity** (agentId
`9313`), this is an autonomous agent with a wallet that *earns per transaction*.

> **✅ Confirmed live on Celo mainnet.** The full round-trip — 402 challenge →
> wallet signature → thirdweb facilitator settlement → 200 + proof pinned — has
> been executed end-to-end. Example settlement:
> [`0xbd66335c…b69350`](https://celoscan.io/tx/0xbd66335c90ac0ea85f6e520697bc4e3bb47053b2effc39bb4ec81c7972b69350)
> (0.01 cUSD, payer → facilitator → agent).

## Flow

```
POST /api/proof
        │
        ▼
  enforceX402()  ── X402 disabled ─────────────► pin proof (free)
        │
   402 Payment Required  (price, asset cUSD, network celo, payTo agent)
        │
        ▼
  client wallet signs the cUSD EIP-712 authorization   (thirdweb/x402)
        │
   retry with PAYMENT-SIGNATURE header
        │
        ▼
  thirdweb facilitator verifies + settles on-chain  →  200  →  pin proof
```

- **Asset:** cUSD / USDm (`0x765DE816845861e75A25fCA122bb6898B8B1282a`) on **Celo mainnet** (42220).
- **Price:** 0.01 cUSD per verification (configurable).
- **payTo:** the agent operator `0x7273dE…3389` (where cUSD lands).
- **Facilitator:** thirdweb (the [Celo-recommended](https://docs.celo.org/build-on-celo/build-with-ai/x402) x402 facilitator; Celo isn't in the default x402 stablecoin map, so thirdweb's facilitator is required).
- **Settlement:** Mento's StableTokenV2 supports **EIP-2612 `permit`**; the facilitator settles via a signed permit (no Permit2 pre-approval needed).

## Two details that make Celo cUSD work

cUSD is not a Coinbase USDC, so two things had to be pinned down against the live
facilitator (both verified, not guessed):

1. **EIP-712 header name.** x402 **v2** carries the signed payment in the
   **`PAYMENT-SIGNATURE`** request header (v1 used `X-PAYMENT`). The server gate
   reads both (`PAYMENT-SIGNATURE` first).
2. **EIP-712 domain.** cUSD was renamed cUSD → USDm; its on-chain
   `DOMAIN_SEPARATOR` (`0x87dc…2f97`) reconstructs from **name `"Mento Dollar"`,
   version `"3"`** (standard full domain, no salt) — *not* version "1"/"2".
   Confirmed by reconstructing the separator on-chain. These are env-overridable
   (`X402_CUSD_NAME` / `X402_CUSD_VERSION`) so a future Mento upgrade needs no
   code change.

Both sides use **thirdweb's x402 client** so the client and server speak the same
dialect. The browser client wraps the connected **wagmi** wallet
(`useWalletClient`) in a thirdweb wallet adapter — the wallet only ever signs
typed data; no thirdweb Connect modal, no Permit2 approval.

## Safety: default-off + fail-open

x402 is **off by default** and **fail-open**: if `X402_ENABLED` is unset, or the
thirdweb keys / client id are missing, or the facilitator errors, the proof
upload proceeds free. It can never break the claim flow or the live demo. The
heavy `thirdweb` SDK is server-only on the API side (`serverExternalPackages`)
and dynamically imported on the client only when x402 is on — neither bloats the
default bundle.

## Enable it

1. In the **thirdweb** dashboard, in your project, get:
   - **Project Secret Key** (unprefixed ~86-char string) — Project → Settings.
   - **Client ID** (public) — shown next to the secret key. It equals the first
     32 hex chars of `sha256(secretKey)`, so it can also be derived offline.
   - A **Server Wallet** (Wallets → Server Wallets) — copy its address, fund with
     ~0.5 CELO for settlement gas.
   - A **Vault Access Token** (`vt_act_…`) — authorizes the server wallet to sign.
     (The `sa_adm_…` admin key is for managing tokens only — keep it offline.)
2. Set in the app env (Railway/Vercel):
   ```bash
   X402_ENABLED=true
   NEXT_PUBLIC_X402_ENABLED=true
   THIRDWEB_SECRET_KEY=...                 # unprefixed Project Secret Key
   NEXT_PUBLIC_THIRDWEB_CLIENT_ID=...      # public Client ID (browser auto-pay)
   THIRDWEB_VAULT_ACCESS_TOKEN=vt_act_...
   X402_SERVER_WALLET_ADDRESS=0x...
   X402_PAY_TO=0x7273dE585311a5139Ef83f0F6Dbb29F3e57b3389
   X402_RESOURCE_URL=https://trywarrant.xyz/api/proof
   ```
3. A claimant submitting a proof signs a single cUSD authorization in their
   wallet; the facilitator settles it on-chain and the proof pins.
